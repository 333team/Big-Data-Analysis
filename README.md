# Big-Data-Analysis

### 1. 擷取訊息 (Retrieve Information)
*   **定義**：這是最基礎的閱讀能力。指讀者能從文章中**直接找到**明確寫出的特定資訊、數據、人名、時間或地點。
*   **難度**：⭐⭐（基礎）
*   **例子**：文章寫「小明早上八點出門」，題目問「小明幾點出門？」。學生只要眼力好、能對應關鍵字就能答對。
*   **數據意義**：如果這個分數低，代表學生可能缺乏耐心，或基本的閱讀掃描能力不足。

### 2. 廣泛理解 (Broad / Global Understanding)
*   **定義**：指讀者讀完文章後，能掌握**全文的主旨**、大意或整體的氣氛。不是看細節，而是看「整片森林」。
*   **難度**：⭐⭐⭐（中等）
*   **例子**：題目問「這篇文章主要在告訴我們什麼道理？」或「請選出這段落的最佳標題」。
*   **數據意義**：如果這個分數低，代表學生可能容易迷失在細節裡，抓不到重點（見樹不見林）。

### 3. 發展解釋 (Develop Interpretation)
*   **定義**：這屬於較高層次的閱讀能力。答案**沒有直接寫在文章裡**，讀者需要結合文章中的多個線索，或運用自己的知識進行**推論（Inference）**，讀出「字裡行間」隱含的意思。
*   **難度**：⭐⭐⭐⭐（進階）
*   **例子**：文章寫「他看著窗外的大雨嘆了口氣，把球鞋收回櫃子」，題目問「他為什麼嘆氣？」（答案推論：因為下雨不能出去打球）。
*   **數據意義**：這是分辨高分群學生的關鍵指標。分數低代表學生缺乏邏輯推理能力，只能理解表面文字。

### 4. 文本形式 (Reflect on and Evaluate Content and Form)
*   **定義**：這是後設認知（Thinking about thinking）的能力。讀者不只是讀內容，還要評估文章的**寫作手法、結構、修辭**或作者的寫作目的。
*   **難度**：⭐⭐⭐⭐⭐（高階）
*   **例子**：題目問「作者為什麼要在這一段使用『排比』修辭？」、「這篇文章引用數據的目的是什麼？」或「這篇文章的結構是如何安排的？」。
*   **數據意義**：這代表學生能否跳脫故事本身，以評論者的角度分析文章。

### 5. 文本理解 (Text Understanding)
*   **註解**：這個項目在不同的測驗定義中可能略有不同，但在您的資料集中，它通常是指**針對文章內容深層含義的理解**，有時是用來統稱「廣泛理解」與「發展解釋」的綜合能力，或者是特指對「文本內容（What）」的理解，以區別於「文本形式（How）」。
*   **定位**：通常介於「廣泛理解」與「發展解釋」之間。
*   **數據意義**：代表學生是否真的「讀懂」了這篇文章在說什麼故事或傳達什麼知識。

---

### 📊 在數據分析中的應用建議


1.  **U型曲線差異**：
    *   **「擷取訊息」**（找事實）這類記憶型題目，遺忘速度通常較快，但也許容易透過短期複習恢復，曲線可能波動大。
    *   **「發展解釋」**（推論）這類理解型題目，一旦學會了通常比較不容易忘，曲線可能較平緩，或者在「適度困難」的延遲後表現更好（因為大腦有時間消化）。

2.  **學生診斷**：
    *   如果某學生「擷取訊息」很高分，但「發展解釋」很低分，代表他是**死記硬背型**的學生。
    *   如果某學生「文本形式」很高分，代表他的**語文敏銳度**很高，適合往文科深造。

這些欄位是教育大數據中非常核心的特徵，能用來精準描繪學生的學習樣態。
